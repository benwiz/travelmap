# travelmap — Agent Guide

## What this is

Ben's personal travel map at [benwiz.com/travelmap](https://benwiz.com/travelmap). Tracks every place he's visited on an interactive map. The data lives in `locations.yml` (1100+ entries); the front-end has two implementations: a 2D Google Maps view (`2d/`) and a 3D globe (`3d/`). The production `index.html` currently embeds an ArcGIS app.

## Repo layout


## Deployment

Push to `master` → GitHub Pages auto-deploys. No build step. The live URL is **https://benwiz.com/travelmap** (custom domain via GitHub Pages).

## Adding locations

Edit `locations.yml`. Format:

```yaml
locations:
  - name: City Name
    lat: 37.8030
    lng: -122.4275
```

After editing, regenerate the CSV if needed:

```bash
bash yml-to-csv.sh
```

## Development

```bash
# Serve locally (any static server works)
http-server
open http://localhost:8080

# Or with Python
python3 -m http.server 8080
```

The 2D view (`2d/`) reads `../locations.yml` relative to its directory. The 3D view (`3d/`) uses its own data files for elevation/population overlays.

## Notes for agents

- **No build step** — all files are served as-is. Don't introduce a bundler without discussing it first.
- **`locations.yml` is the source of truth** — if adding/removing places, always edit here and regenerate the CSV.
- **Commits to `master` go live immediately** — be deliberate with what you push.
- The ArcGIS embed in `index.html` (appid `14f2f6bce05a4eacbb242c359bd7b8ea`) is a separate hosted app; to change the map data there you'd update the ArcGIS app directly, not the files here.
